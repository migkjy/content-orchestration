## 1. 플랫폼 목표

멀티 프로젝트 콘텐츠 오케스트레이션 — 여러 캠페인(PLF 등)의 콘텐츠를 채널별/일정별로 계획하고 발행까지 관리하는 통합 플랫폼.

**첫 번째 실제 사용 케이스**: Richbukae 채널 × AI 설계자 세일즈 PLF

---

## 2. 범위 결정 (중요)

| 영역 | 결정 |
|------|------|
| 콘텐츠 파이프라인 (RSS 수집, 아이디어 수집) | **별도 프로젝트로 분리** — 추후 API 연동 |
| 퍼널 마케팅 이메일 (단계별 자동 발송) | **별도 프로젝트로 분리** (Resend 또는 Brevo) |
| 콘텐츠 오케스트레이션 (발행 계획 + 실행) | **현재 집중 영역** |

기존 content-orchestration의 RSS 수집, 파이프라인 로그 화면은 오케스트레이션 범위 밖.

## 3. 핵심 개념 & 데이터 모델

### 3.1 프로젝트 (Project / Campaign)
- **정의**: 목적 있는 콘텐츠 묶음 (PLF 캠페인, 브랜딩 캠페인 등)
- **타입**:
  - 캠페인형 (시작~종료 날짜 있음 — AI 설계자 시리즈 등)
  - 상시형 (날짜 없음 — 지속 발행)
- **속성**: id, name, description, goal, start_date(optional), end_date(optional), status

### 3.2 채널 (Channel)
- **정의**: 콘텐츠를 발행하는 플랫폼 + 계정
- **종류**: 인스타그램, 유튜브, 뉴스레터, 블로그, 페이스북, X 등
- **속성**: id, platform, account_name, connection_type, connection_status, connection_detail
- **연동 방식**:
  - SNS (인스타, 페이스북, X): GetLate
  - 뉴스레터: Brevo API
  - 블로그: WordPress API
  - 유튜브: 수동 (또는 YouTube API)

### 3.3 콘텐츠 슬롯 (Content Item)
- **핵심 개념**: 계획(슬롯) → 실행(콘텐츠 작성)이 동일한 아이템
- **속성**: id, project_id, channel_id, title, content_body, scheduled_at, status, rejected_reason
- **상태 플로우**:
  ```
  미작성 → Draft → 검토요청 → 반려(수정지시 코멘트) → Draft (재수정)
                            ↓
                         승인완료 → 예약 → 발행완료
                         취소 (어느 단계에서든 가능)
  ```

## 4. 화면 구조 (4개 뷰)

### 4.1 홈 화면 `/`

프로젝트 카드 목록. 상태 중심 표시.

```
콘텐츠 오케스트레이션          [+ 새 프로젝트]
──────────────────────────────
[AI 설계자 세일즈 PLF]  ● 진행중
 채널 4개  Draft 5 · 예약 3 · 발행완료 누적 10건
                              [열기]

[Richbukae 브랜딩 캠페인]  ● 준비중
 채널 3개  Draft 6 · 예약 0 · 발행완료 누적 0건
                              [열기]

[+ 새 프로젝트 만들기]
──────────────────────────────
빠른 탐색: [채널별 뷰] [캘린더]
```

- Draft+예약 = 지금 진행 중인 것 (항상 최신)
- 발행완료 = 누적 총 수 (참고용)

### 4.2 프로젝트 상세 `/projects/[id]`

콘텐츠 슬롯 기반. 채널별/상태별 그룹핑 + 필터.

**상단**: 프로젝트 목표 + 채널별 계획 달성 현황

```
← AI 설계자 세일즈 PLF    [편집] [+ 콘텐츠 추가]
목표: 리치부캐 채널 AI 설계자 서비스 세일즈 전환

채널별 달성 현황:
  인스타그램  계획 12건  완료 3건  남은것 9건
  유튜브      계획  4건  완료 1건  남은것 3건
  뉴스레터    계획  3건  완료 0건  남은것 3건
  블로그      계획  6건  완료 2건  남은것 4건
```

**뷰 전환**: [리스트] [칸반] — 그룹: [채널별▼] / 필터: [상태▼] [기간▼]

**채널별 그룹 리스트**:
```
📱 인스타그램 @richbukae_official (12계획 · 3완료 · 9남음)
 3/5  09:00  AI 설계자란?     ● 발행완료  [보기]
 3/7  09:00  설계자 되는 법   ● 예약      [편집]
 3/10 09:00  [빈 슬롯]        ○ 미작성    [작성하기]
```

**핵심**: 빈 슬롯 [작성하기] 클릭 → 즉시 에디터 진입 (플랜→실행 단절 없음)

### 4.3 채널별 뷰 `/channels`

채널 목록 + 연동 상태. 각 채널 클릭 시 프로젝트별 그룹핑.

**채널 목록**:
```
채널 & 계정 관리           [+ 채널 추가]
──────────────────────────────────────────
📱 인스타그램 @richbukae_official
  연동: GetLate ✅ 자동발행 가능
  Draft 3 · 예약 2 · 발행완료 8   [콘텐츠 보기]

📰 뉴스레터 contact@richbukae.com
  연동: Brevo API ✅ 자동발행 가능
  Draft 2 · 예약 1 · 발행완료 5   [콘텐츠 보기]

🎥 유튜브 @richbukae
  연동: ⚠️ API 미연결 (수동 발행)

✍️ 블로그 richbukae.com/blog
  연동: WordPress API ✅ 자동발행 가능
```

**채널 상세 (프로젝트별 그룹)**:
```
📱 인스타그램 @richbukae_official
그룹: [프로젝트별▼]  필터: [상태▼]
──────────────────────────────────────
📌 AI 설계자 세일즈 PLF (8건)
  3/5  AI 설계자란?     ● 발행완료
  3/10 [빈 슬롯]        ○ 미작성   [작성하기]

📌 Richbukae 브랜딩 캠페인 (3건)
  3/8  부자의 습관      ● Draft
```

그룹 전환: [프로젝트별▼] ↔ [계정별▼]

**연동 상태**: ✅ 자동발행 가능 / ⚠️ 미연결 / ❌ 연결오류

### 4.4 캘린더 뷰 `/calendar`

월/주 캘린더. 채널별 아이콘으로 시각화.

```
캘린더                  [< 2월] [3월 2026] [4월 >]
필터: [전체 프로젝트▼] [전체 채널▼]
──────────────────────────────────────────────
월    화    수    목    금    토    일
 3     4     5     6     7    8     9
       📱2   📱1   🎥1   📰1  ✍️1
```

날짜 클릭 → 해당 날짜 콘텐츠 팝업 (편집/취소 가능)

아이콘 범례: 📱인스타  🎥유튜브  📰뉴스레터  ✍️블로그

## 5. 기존 코드 활용 전략

- `components/content-editor.tsx` — 콘텐츠 에디터 재사용
- `app/[project]/content/` — 콘텐츠 CRUD 로직 재사용
- `lib/content-db.ts` — DB 쿼리 함수 재사용
- **신규 테이블 추가**: `campaigns` (프로젝트), `channels` (채널+연동 정보)
- **기존 `content_items` 테이블**: `campaign_id`, `channel_id` 컬럼 추가

## 6. 확정 제외 항목 (향후 별도)

- RSS/파이프라인 수집 화면 (별도 프로젝트)
- 퍼널 마케팅 이메일 자동화 (Resend/Brevo 별도 프로젝트)
- 자동 발행 실행 엔진 (API 연동 구현)
